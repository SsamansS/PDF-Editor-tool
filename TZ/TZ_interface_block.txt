ТЗ: PDF Editor v2 — финальная версия
Стек
PDF.js — клиентский рендеринг
Vanilla JS — overlay, логика
Python HTTP server — API (расширяем существующий)
Флоу пользователя

1. Открыть PDF → PDF.js рендерит страницу
2. Hover на блок → подсвечивается весь текстовый блок
3. Клик → в правой панели появляется textarea с текстом блока
4. Редактируем текст
5. "Применить" → заменяется ТОЛЬКО этот конкретный блок по позиции
6. PDF.js перезагружает страницу → видим результат
7. "Отмена" (Undo) → откатывает последнее действие
Компоненты UI
Левая часть — viewer

PDF.js canvas
Прозрачный <div> overlay поверх canvas — хайлайты блоков
Навигация, zoom
Правая панель

Textarea с текстом выбранного блока
Кнопки: Применить / Отмена / Undo
Лента изменений: список того что было заменено (с возможностью undo по каждому)
Backend API
Endpoint	Метод	Параметры	Что делает
/serve?path=...	GET	путь к PDF	отдаёт PDF файл для PDF.js
/api/blocks?path=...&page=N	GET	путь, страница	возвращает все блоки с bbox и текстом
/api/replace	POST	path, page, block_bbox, old, new	заменяет ТОЛЬКО этот блок
/api/undo	POST	path	откатывает последнее действие
Undo — механизм
Самый надёжный подход — хранить оригинал + список изменений:


original.pdf  (никогда не трогаем)
edit_history = [
  {page: 2, bbox: [...], old: "Desc", new: "Aes"},
  {page: 3, bbox: [...], old: "Stripe", new: "Bank"},
]

Undo → удаляем последний элемент из edit_history
       → пересобираем _edited.pdf из original + оставшихся изменений
Это гарантирует корректность даже при нескольких undo подряд.

Файловая система

document.pdf          ← оригинал, только чтение
document_edited.pdf   ← рабочий файл, перезаписывается при каждом изменении
Открытый технический вопрос
Замена по позиции (а не по тексту везде) требует нового backend-метода:

вместо replace_text("Desc", "Aes") → replace_at_bbox(page=2, bbox=[84,120,103,129], old="Desc", new="Aes").

Это реализуемо через page.search_for(old, clip=bbox) — ищем только внутри этой области. Технически чуть сложнее, но решаемо.
